## Task 4: Secrets logging verification (verify-only)

App secrets are already `SecretStr` (`OPENROUTER_API_KEY`, `MINIO_SECRET_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_JWT_SECRET`, `SENTRY_DSN`). `DATABASE_URL` is a `PostgresDsn` (kept as-is). This task confirms none of them leak into logs.

**Files:** none expected (verification). Only edit if a leak is found.

- [ ] **Step 1: Grep for any log/print that interpolates a secret or the DB URL**

```bash
cd /Volumes/External/promptly/qa-chatbot
grep -rniE "log\.(info|warning|error|debug|exception)\(.*(DATABASE_URL|effective_url|password|secret|service_role|jwt_secret|api_key|token)" src --include='*.py' | grep -viE "user_id|job_id|api_key_repo|key_hash|hashed|set_job|api_key\.id|invalid|missing|token=token\b"
echo "--- print() of secrets/URLs? ---"
grep -rniE "print\(.*(DATABASE_URL|effective_url|password|secret|service_role|api_key)" src --include='*.py'
echo "--- f-strings embedding effective_url / DATABASE_URL ---"
grep -rn "effective_url\|DATABASE_URL" src --include='*.py'
```
Expected: no log/print line that emits a secret value. (`effective_url`/`DATABASE_URL` should appear only where the engine/alembic is constructed — `db/session.py`, `migrations/env.py`, `config/database.py` — never inside a log call.)

- [ ] **Step 2: Confirm `SecretStr` coverage of secret-bearing settings**

```bash
cd /Volumes/External/promptly/qa-chatbot
grep -rn "SecretStr" src/app/config src/app/llm/settings.py
```
Expected: `OPENROUTER_API_KEY`, `MINIO_SECRET_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_JWT_SECRET`, `SENTRY_DSN` all typed `SecretStr`.
